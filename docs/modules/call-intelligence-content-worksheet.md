# Call Intelligence — Content Verification Worksheet

> **Purpose:** the launch content for `service_cases` + `conversation_hooks`, converted from the
> retired engineering seed (`supabase/seeds/call_intelligence_seed.sql`, deleted 2026-06-12 —
> decision: nothing is bulk-seeded). **Every row below is a DRAFT written by engineering to the
> §13 content brief. None of it is verified delivery history yet.**
>
> **Process (per row):**
> 1. Set **Status** to one of: `VERIFIED` (claim matches a real delivery — name the internal
>    reference), `EDITED` (rewrite the row to match what actually happened, then verify), or
>    `REJECTED` (no real delivery backs it — the row dies).
> 2. Fill **Ref / Owner** — who verified it and against what (client file, deal record, trip ID).
> 3. Only `VERIFIED` rows are entered — **via the admin path** (`upsertServiceCaseAction` /
>    `upsertConversationHookAction`, or the Supabase dashboard as admin). **Never bulk-SQL.**
>
> **Field rules (docs/modules/call-intelligence.md §13):** title ≤120 chars, claim not description;
> summary 2–3 sentences, past tense, NO client names; city slug always in tags (lowercase,
> underscores, ≤10 tags); 2 featured per category; domain for all rows: `onboarding`.
>
> Target: ≥20 verified per category (spec ship bar). UI demo allowed at ≥10 per category.
